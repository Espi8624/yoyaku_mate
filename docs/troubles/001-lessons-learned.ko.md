## CORS 및 Vercel 프록시 설정

로컬과 프로덕션 환경에서 발생하는 CORS 문제를 해결하기 위해 Vercel의 Rewrite 기능을 프록시로 설계했습니다.  
환경에 따라 API_BASE_URL을 달리 설정하여 개발/프로덕션을 깔끔하게 분리할 수 있었습니다.

→ [ADR-001: Vercel Rewrite 프록시 채택](../decisions/ADR-001-vercel-proxy.ko.md)

## 네트워크 오프라인 상태 대응 (SSE 자동 재연결)

대기 중인 손님이 일시적인 통신 음영 지역 등으로 오프라인 상태가 되어도, `EventSource`의 자동 재연결 메커니즘이 백그라운드에서 동작합니다.  
온라인 복귀 직후 서버의 호출 알림이나 입장 완료 통지를 누락 없이 수신할 수 있습니다.

서버는 SSE 연결 즉시 현재 상태를 초기 데이터로 전송하므로, 재연결 후 별도 상태 복원 로직 없이도 자동으로 동기화됩니다.

→ [SSE 클라이언트 구현](../implementation/sse-client.ko.md)

## AI 컨텍스트 프롬프팅 최적화

외부 LLM을 웹 서비스에 연동할 때, 질문 시점의 실시간 점포 대기 상황과 기본 설정 데이터를 System Prompt에 주입하면 환각(Hallucination)을 크게 줄일 수 있습니다.

고정된 정보(메뉴, 위치)와 실시간 정보(현재 대기 수, 예상 시간)를 분리하여 Prompt를 구성하면 응답의 정확도와 품질이 향상됩니다.

## SSE 선택 근거 (WebSocket 대비)

WebSocket 대비 오버헤드가 적고, HTTP 프로토콜 위에서 동작하여 Vercel 프록시 환경과의 호환성이 높습니다.  
대기열 데이터는 서버 → 클라이언트 단방향으로만 전달하면 되므로 SSE로 충분합니다.
